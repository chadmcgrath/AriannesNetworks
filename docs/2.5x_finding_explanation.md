# The 2.5x Finding: Item Aggregation vs Sample Size Effects

## Key Research Finding from Herrera-Bennett & Rhemtulla (2021)

### The 2.5x Improvement Finding

**Core Finding:** Increasing the number of items per facet in a survey can enhance network centralization metrics to a degree comparable to increasing the sample size by 2.5 times.

### Specific Examples:

- **Original sample size 84** → 2.5x increase = **210 samples**
- **Original sample size 212** → 2.5x increase = **530 samples**

### What This Means:

Augmenting the number of items per facet in a survey can yield improvements in **centralization metrics** equivalent to those achieved by significantly increasing the sample size.

### Implementation in Our Code:

The finding suggests that:
1. **Item Aggregation Effect**: Going from 1 item per facet to 2 items per facet
2. **Sample Size Effect**: Increasing sample size by 2.5x (e.g., 84 → 210, or 212 → 530)
3. **Equivalent Improvement**: These two effects should produce similar improvements in network centralization

### What We Should Be Comparing:

- **Aggregation improvement**: 1 item per facet vs 2 items per facet (at same sample size)
- **Sample size improvement**: Small sample vs 2.5x larger sample (with same aggregation level)
- **Centralization metrics**: The key measure is network centralization, not global strength
